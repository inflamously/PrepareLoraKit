# Complexity Technical Debt

Audited on 2026-07-11. This list highlights first-party files that exceed roughly 300 lines of code, carry too many responsibilities in one place, or are otherwise harder than necessary to read and modify.

The scan excludes dependency/build/cache directories, the `third_party/seedvr2` submodule, and repository skill tooling under `.claude/` and `.codex/`. It also omits `prepare_lora_kit_ui/static/core/api.js` (310 LoC) because inspection showed that it is a cohesive, almost entirely declarative JSDoc contract for the pywebview bridge.

| --- | --- |
| `tests/ui/test_runner.py` | 816 LoC; combines selection and prerequisite validation, project and image payloads, run sequencing and step-config pauses, media-server behavior, interaction payloads, logging, cancellation and job state, region captioning, and dataset annotation tests. Split by runner validation, media serving, job lifecycle, and interaction behavior. |
| `prepare_lora_kit/steps/upscale/step.py` | 698 LoC; owns top-level orchestration, candidate partitioning and review, destination/collision rules, upscaler selection, SeedVR2 batching, JPEG cleanup, hallucination rejection, pass-through and temporary-file handling, and report serialization. The planning/review, processing backends, and output/report concerns are separable. |
